## FizzBuzz

```wenyan
為是百遍。
  加一以其。名之曰「i」。
  除「i」以十五。所餘幾何。名之曰「餘」。
  若「餘」等於零者。
    吾有一言。曰「「FizzBuzz」」。書之。
  若非。
    除「i」以三。所餘幾何。名之曰「餘三」。
    若「餘三」等於零者。
      吾有一言。曰「「Fizz」」。書之。
    若非。
      除「i」以五。所餘幾何。名之曰「餘五」。
      若「餘五」等於零者。
        吾有一言。曰「「Buzz」」。書之。
      若非。
        夫「i」。書之。
      也。
    也。
  也。
云云。
```

## 斐波那契

```wenyan
吾有一術。名之曰「斐波那契」。欲行是術。必先得一數。曰「n」。乃行是術曰。
  若「n」不大於一者。
    乃得「n」。
  也。
  減「n」以一。名之曰「甲」。
  減「n」以二。名之曰「乙」。
  施「斐波那契」於「甲」。名之曰「丙」。
  施「斐波那契」於「乙」。名之曰「丁」。
  加「丙」以「丁」。乃得矣。
是謂「斐波那契」之術也。

施「斐波那契」於十。書之。
```

## 列操作

### 建與遍歷

```wenyan
吾有一列。名之曰「數列」。
充「數列」以一。以二。以三。以四。以五。

凡「數列」中之「x」。
  乘「x」以「x」。書之。
云云。
```

### 用標準庫（列經）

```wenyan
吾嘗觀「「列經」」之書。方悟「遍施」「篩剔」「排序」之義。

吾有一術。名之曰「大於三」。欲行是術。必先得一數。曰「x」。乃行是術曰。
  若「x」大於三者。乃得陽也。若非。乃得陰也。
是謂「大於三」之術也。

施「篩剔」於「大於三」於「數列」。名之曰「結果」。
```

## 言操作

言為基本型。拼接用`銜`，複雜操作需自定術或JS互操。

```wenyan
吾有一言。曰「「天地」」。名之曰「甲」。
吾有一言。曰「「玄黃」」。名之曰「乙」。
銜「甲」以「乙」。名之曰「丙」。書之。
注曰「「丙 = 天地玄黃」」
```

## 遞歸術

式：定術，體內以名自調。

```wenyan
吾有一術。名之曰「階乘」。欲行是術。必先得一數。曰「n」。乃行是術曰。
  若「n」不大於一者。
    乃得一。
  也。
  減「n」以一。名之曰「m」。
  施「階乘」於「m」。名之曰「r」。
  乘「n」以「r」。乃得矣。
是謂「階乘」之術也。
```

## 模塊式

### 建模塊（mylib.wy）

```wenyan
注曰「「私有用吾有」」
吾有一數。曰零。名之曰「計數」。

注曰「「公開用今有」」
今有一術。名之曰「增」。是術曰。
  加「計數」以一。昔之「計數」者。今其是矣。
  乃得「計數」。
是謂「增」之術也。
```

### 用模塊

```wenyan
吾嘗觀「「mylib」」之書。方悟「增」之義。
施「增」。書之。
施「增」。書之。
```

## 錯誤處理式

### 簡式全捕

```wenyan
姑妄行此。
  ⋯⋯
如事不諧。不知何禍歟。名之曰「e」。
  夫「e」之「「名」」。書之。
  夫「e」之「「文」」。書之。
乃作罷。
```

### 拋自定義錯誤

```wenyan
嗚呼。「「自定義」」之禍。曰「「出了問題」」
```

### 錯誤屬性

捕獲之錯誤有：
- `「「名」」`→錯誤名
- `「「文」」`→錯誤訊息
- `「「蹤」」`→堆棧追蹤

## 常見陷阱

### 1. 列1起始
```wenyan
吾有一列。名之曰「a」。充「a」以十。以二十。以三十。
夫「a」之一。書之。    注曰「「印10（首元素）」」
夫「a」之二。書之。    注曰「「印20（次元素）」」
```

### 2. 字串vs變量括號
- `「「text」」`→字串字面量（雙角括號）
- `「varname」`→變量引用（單角括號）

### 3. 塊終止符
- `也`→結`若...者`及`若非`塊
- `云云`→結`為是...遍`、`恆為是`、`凡...中之`循環
- `是謂...之術也`→結術定義

### 4. 以vs於
- `加X以Y`→X+Y
- `加X於Y`→Y+X（於反序！）

### 5. 其清棧
`其`引最近無名結果且清棧。勿連用多`其`：
```wenyan
注曰「「劣：加其以其 易混」」
注曰「「優：命名中間結果」」
加一以二。名之曰「temp」。加「temp」以三。
```

### 6. 標點可省
皆合法：
```wenyan
吾有一數。曰三。名之曰「甲」。
吾有一數曰三名之曰「甲」
```
然加標點大益可讀。

### 7. 今有vs吾有
- `今有`→公/導出（導入者可用）
- `吾有`→私/域內

模塊API用`今有`，內部狀態用`吾有`。

## 數字面量

文言用漢字數：

| 文言 | 值 |
|---|---|
| `零` | 0 |
| `一` | 1 |
| `二` | 2 |
| `三` | 3 |
| `四` | 4 |
| `五` | 5 |
| `六` | 6 |
| `七` | 7 |
| `八` | 8 |
| `九` | 9 |
| `十` | 10 |
| `百` | 100 |
| `千` | 1000 |
| `萬` | 10000 |
| `億` | 100000000 |

組合：`三百二十一`=321，`一萬二千`=12000

小數：`三點一四`=3.14

數碼式：`二〇二〇`=2020（逐位讀）

## 編譯目標

```bash
npx wenyan -c file.wy           # →JS（默認）
npx wenyan -c -l py file.wy     # →Python
npx wenyan -c -l rb file.wy     # →Ruby
npx wenyan -c --roman pinyin file.wy  # 羅馬化標識符
```

目標差異：
- JS：全特性，輸出列0起始
- Python：多數特性支持，部分stdlib僅JS
- Ruby：多數特性支持，部分stdlib僅JS
